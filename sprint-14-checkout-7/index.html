workshop sveltekit principes over best practices in sveltekit

tijdens de workshop heb ik error handler van sveltekit uitgezocht en een gist geschreven

leervragen

  • Welke stappen neem je om een project in SvelteKit op te starten?
    • een site map maken
    • in de home route +page.svelte, +layout.svelte en +layour.server.js files maken
    • nadenken over welke paginas een dynamische route nodig hebben, en ook passende namen bedenken daarvoor
    • en ook alvast +page.svelte aanmaken in elke map
  • Hoe kan je AI gebruiken om principes en best-practices van een framework te ontdekken?
    • door eerst best practices in svelte op te zoeken
    • daarna per "practice" meer informatie opzoeken
  • Welke SvelteKit principes en best-practices heb je geleerd te gebruiken?
    • splitting code: maak componenten van code die je vaker gaat gebruiken
    • +error.svelte: svelte heeft een ingeboude error handler om een error pagina te maken
    • bind: om variables te sturen naar een parrent file/folder
    • props: om data door te sturen naar componenten
